
🛡️ ProQuant 旗艦戰情室 (原：策略回測系統) v2.0
版本追蹤： 2026.03.23 (v2.0 旗艦升級版)

系統定位： 專為量化設計的期貨多策略風險防禦與邏輯診斷中心。不僅提供實戰部位模擬，更結合法人級別的資金控管邏輯，協助交易者在「管理成本」與「資金擴張」間取得完美平衡。

🚀 v2.0 核心更新大綱 (What's New)
1. 🎨 視覺與 UI 全面重構 (Blue-Purple Tech Style)
SaaS 級現代化介面：全面升級為「明亮白底 + 藍紫科技漸層」風格，並導入顯著的圓角卡片 (16px) 與輕度浮雕陰影設計，大幅提升長時間盯盤的視覺舒適度與專業感。

互動式圖表升級：Plotly 淨值圖表改為科技漸層藍主線，搭配半透明紅色的回撤警戒區間，並支援點擊圖例自由切換單一策略疊加對比。

2. 🧠 法人級「風險防禦」資金配置法
捨棄傳統單純的「保證金槓桿」，導入更嚴謹的底層邏輯：系統建議資金 = 原始保證金 + (歷史最大 MDD × 風險防禦倍數)。

雙資金維度對比：支援使用者手動輸入「目前可用總資金」，系統會自動與「科學建議資金」進行比對，計算真實的資金使用率與名目槓桿。

3. 🎯 策略「生命力」診斷指標 (Vitality Metrics)
不再只看損益數字，新增四大底層邏輯診斷指標，抓出過度擬合或市場適應不良的策略：

獲利因子 (Profit Factor)：衡量賺賠效率。

勝率 (Win Rate)：監控市場節奏。

連續虧損次數 (Max Consecutive Losses)：偵測極端逆風期。

均恢復期與目前套牢天數：計算時間成本與資金停滯風險。

4. ⚖️ 核心數據 100% 絕對對齊
結算級 MDD 運算：修正了每日加總造成的誤差，現在「組合歷史最大回撤」與「綜合夏普值」皆採用逐筆結算 (Transaction-based) 與日曆補齊對齊，確保單一策略與組合檢視時的數值完美一致。

0 口數動態排除：支援實戰口數設定為 0，系統將自動過濾該策略且不會產生 NaN 運算錯誤。

📘 系統指標定義與風險管理指南
為確保操作紀律，本系統內建以下兩大核心防禦燈號機制：

🚦 一、 策略壓力紅綠燈 (單一策略有效性監控)
監控單一策略是否偏離其歷史統計特性，作為「汰弱留強」的客觀依據：

🟢 穩定運行：目前回撤 < 歷史 MDD 的 80%。策略表現符合預期，可安心執行。

🟡 性能鈍化：目前回撤達 80%~100%，或「目前套牢天數」>「均恢復期的 2 倍」。代表策略獲利效率正在下降（資金卡住太久），需密切觀察。

🟠 進入未知：目前回撤達 100%~150% (破底期)。策略遇到歷史未見的逆風，建議減碼或暫停新倉。

🔴 策略失效：目前回撤 > 150%。嚴重偏離歷史規律，系統判定邏輯已失效，建議強制停機。

 二、 組合資金安全狀態 (總帳戶抗風險能力)
根據您輸入的「實際可用資金」與系統計算出的「建議資金」進行階梯式安全判定：

🛡️ 極度安全 (Robust)：總資金 > 建議資金的 1.5 倍。足以承受極端黑天鵝事件。

✅ 資金充裕 (Safe)：總資金 >= 系統建議資金。符合您設定的風險防禦門檻，資金效率最佳。

⚠️ 緩衝不足 (Warning)：總資金 < 建議資金，但仍大於 (組合 MDD + 基本保證金)。容錯空間極低，一旦發生歷史級回撤將面臨追繳警告。

🚨 斷頭風險 (Danger)：總資金 < 組合 MDD + 基本保證金。資金水位嚴重不足，發生歷史回撤時帳戶將直接斷頭。

🕹️ 使用說明 (Quick Start)
匯入策略：從左側面板上傳 TradingView 匯出的交易清單 (.csv 或 .xlsx)。

資金與防禦設定：

輸入您準備用於實戰的「目前可用總資金」。

設定「風險防禦倍數」(建議值為 1.5 ~ 2.0，代表您希望帳戶能承受幾次歷史最大回撤)。

商品與口數配置：為每個上傳的策略選擇對應的期貨合約商品（支援大台、小那、日經等自動換算），並調整「實戰口數」（設為 0 即排除該策略）。

執行診斷：點擊「開始執行邏輯診斷」，戰情室將自動生成 8 大綜合指標、資金狀態燈號、風險明細表與淨值對比圖。
